KISS, YAGNI, & DRY Codebase Audit

Initial findings focused on the Admin module and Server Actions

1. Redundansi Route & Direktori (Admin)

Ada beberapa direktori yang tampak redundan atau tidak konsisten:

  • admin/categories/[id]/edit VS admin/categories/edit/[id] (DRY Violation)
  • Struktur posts juga memiliki pola serupa.

2. Inkonsistensi Data Fetching (KISS Violation)

  • Categories: Menggunakan services/data-service.ts di Server Component.
  • Posts: Menggunakan actions/posts.ts dengan useEffect di Client Component.

Rekomendasi: Standarisasi menggunakan Server Components untuk list data guna performa lebih baik (KISS).

3. Duplikasi Skema Validasi (DRY Violation)

  • categorySchema didefinisikan di actions/categories.ts DAN validations/index.ts.
  • Rekomendasi: Konsolidasikan semua skema ke satu tempat (lib/validations).

Refaktor Server Actions

B

Standarisasi Admin Pages

Gunakan Server Components & bereskan struktur route (KISS/DRY).

Pembersihan Services & Utils
