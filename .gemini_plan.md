# Feature Plan: Resident Data Completeness Audit (Lengkapi Data)

## Problem Statement
Administrators need a way to identify and fix resident records that have missing or "null" data, specifically grouped by sub-village (dusun), to improve the quality of village demographics.

## Goal
Provide a user-friendly dashboard that highlights data gaps per dusun and allows for quick access to fixing those records.

## Completeness Criteria
A record is considered **incomplete** if any of these fields are null, empty, or contain placeholder values (e.g., "-", "N/A"):
- `nik` / `nik_enc`
- `kk` / `kk_enc`
- `name` / `name_enc`
- `birth_place`
- `birth_date`
- `gender`
- `education`
- `occupation`
- `marital_status`
- `father_name`
- `mother_name`
- `dusun`
- `data_year`
